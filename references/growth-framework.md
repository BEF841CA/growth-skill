# 认知成长提取方法论

本文档定义 growth-skill 的核心方法论：如何从日常观察中提取认知信号，如何验证和演化认知模式。

---

## 一、信号类型定义

| 信号类型 | 说明 | 示例 |
|---------|------|------|
| `mental_model_seed` | 可能的心智模型雏形 | "遇到问题先自己试" |
| `decision_heuristic_seed` | 决策模式雏形 | "面对选择时总是选更安全的那个" |
| `value_expression` | 价值观表达 | "他觉得分享比独占更快乐" |
| `anti_pattern` | 明确拒绝的行为 | "他非常讨厌被强迫做某事" |
| `expression_fingerprint` | 认知框架偏好 | "他描述事物喜欢用比喻" |
| `contradiction` | 与已有信号矛盾 | "以前怕生，现在主动跟人打招呼了" |
| `milestone` | 重要人生事件 | "开始上幼儿园了" |
| `emotional_pattern` | 情绪反应模式 | "受挫时会先沉默再爆发" |

### 信号数据结构

每条信号在 `signals.jsonl` 中为一行 JSON。**更新策略为 append-only**：信号被匹配更新时，不修改旧行，而是追加一行新的完整 JSON（包含更新后的所有字段）。读取时按 `id` 去重，取最新出现的行作为当前状态。月度归档时会压缩为每个信号只保留最新版本。

结构如下：

```json
{
  "id": "sig-[uuid短码]",
  "type": "mental_model_seed",
  "status": "seed",
  "content": "遇到问题先自己试",
  "original_excerpts": [
    {"date": "2026-03-15", "text": "今天碰到不会的拼图，先自己试了5分钟才来找我"},
    {"date": "2026-04-02", "text": "杯子倒了没叫我，自己拿抹布擦"}
  ],
  "occurrence_count": 2,
  "domains": ["学习探索", "行为习惯"],
  "cross_domain_count": 2,
  "first_seen": "2026-03-15",
  "last_seen": "2026-04-02",
  "stability": 60,
  "significance": 1.0,
  "observation_dates": ["2026-03-15", "2026-04-02"],
  "related_signal_id": null,
  "corrected": false,
  "reconsolidated_from": null
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 信号唯一标识，`sig-` + 短随机码 |
| `type` | string | 信号类型，见上方类型定义表 |
| `status` | string | 信号状态：seed / emerging / established / evolving / dormant / forgotten |
| `content` | string | 信号内容描述（自然语言） |
| `original_excerpts` | array | 原文摘录列表，每条包含日期和原文。全部保留，不裁剪 |
| `occurrence_count` | int | 被观察匹配到的次数 |
| `domains` | array | 出现过的领域列表 |
| `cross_domain_count` | int | 跨域计数（不同域的出现次数） |
| `first_seen` | date | 首次出现日期 |
| `last_seen` | date | 最后一次被匹配的日期 |
| `stability` | float | 遗忘曲线稳定性参数（天），初始值取决于信号类型 |
| `significance` | float | 信号固有权重：milestone=1.5, contradiction=1.3, 其他=1.0 |
| `observation_dates` | array | 关联的观察日期列表，用于删除记录时定位信号 |
| `related_signal_id` | string/null | 关联信号 ID（contradiction 类型指向原信号，其他为 null） |
| `corrected` | bool | 是否被用户纠偏修正过 |
| `reconsolidated_from` | string/null | 再巩固来源路径（如 "consolidating/2026-01-summary.md"），null 表示非再巩固信号 |

## 二、信号状态流转

```
seed (出现1次，仅记录)
  ↓ 出现2-3次，跨1+领域
emerging (模式开始浮现，值得关注)
  ↓ 出现3+次，跨2+领域，有预测力
established (已确立的认知模式)
  ↓ 最近出现但描述有变化
evolving (正在演化，可能是认知转变的前兆)
  ↓ 长时间未再出现（超过 stability 天数）
dormant (休眠，不代表消失)
```

**再巩固路径**：dormant 信号被新观察重新匹配 → 回到 working memory → 重新评估状态。

## 三、领域分类

日常观察中的"域"由系统根据观察内容自动标注。以下为参考分类，系统可根据实际情况归纳新域：

| 域 | 说明 | 示例关键词 |
|----|------|-----------|
| 社交互动 | 与他人的关系和行为 | 分享、抢、帮助、合作、冲突 |
| 情绪反应 | 情绪表达和调节方式 | 开心、哭、沉默、爆发、害怕 |
| 学习探索 | 对新知识/新事物的态度 | 试、拆、问为什么、模仿、放弃 |
| 行为习惯 | 重复性的行为模式 | 每次、总是、一定、习惯 |
| 价值观表达 | 对是非、好坏的判断 | 公平、对错、应该、喜欢、讨厌 |
| 身体发展 | 运动和能力发展 | 走、跑、抓、站、画 |
| 语言表达 | 说话方式和内容变化 | 说、问、讲、叫、不 |

**跨域计数规则**：同一认知模式（如"遇到困难先自己试"）在不同域中出现，跨域计数+1。跨域计数是信号从 emerging 升级为 established 的关键指标。

## 四、自适应验证阈值

根据观察数据量动态调整验证严格程度：

| 观察数量 | 跨域阈值 | 出现次数阈值 | 最大模型数 | 置信度上限 | 成熟度 |
|---------|---------|------------|----------|----------|-------|
| < 10 | 1 | 1 | 2 | 0.4 | early |
| 10-50 | 1 | 2 | 4 | 0.6 | developing |
| 50-100 | 2 | 3 | 6 | 0.8 | mature |
| > 100 | 2 | 3 | 7 | 1.0 | mature |

**原则**：数据少时宽容（宁可漏掉也不误判），数据多时严格（宁缺毋滥）。

## 五、遗忘曲线与记忆周期

### 遗忘衰减函数

```
retention = e^(-t / stability)
```

- `t` = 距离最后一次被引用/匹配的天数
- `stability` = 信号的稳定性参数

### 初始稳定性参数

| 信号类型 | 初始 stability（天） | 记忆类比 |
|---------|-------------------|---------|
| `milestone` | 180 | 闪光灯记忆，极度抗遗忘 |
| `contradiction` | 120 | 成长转折点，高显著性 |
| `mental_model_seed` | 60 | 需要反复验证才能巩固 |
| `anti_pattern` | 60 | 反模式一旦形成较稳定 |
| `value_expression` | 45 | 中等稳定性 |
| `decision_heuristic_seed` | 45 | 中等稳定性 |
| `expression_fingerprint` | 30 | 变化较快 |
| `emotional_pattern` | 30 | 随情境变化 |

### 重复增强效应

每次新观察匹配到已有信号时：

```
stability_new = stability_old × (1 + 0.5 × ln(occurrence_count))
```

### 合成时的有效权重

```
effective_weight = significance × retention × recency_boost
```

- `significance`：milestone=1.5, contradiction=1.3, 其他=1.0
- `retention`：遗忘曲线保留率（0-1）
- `recency_boost`：最近30天被引用 ×1.3（近因效应）

### 记忆归档层级

| 层 | 存储 | 行为 |
|---|---|---|
| 工作记忆 | `signals.jsonl` | 容量~200条，保留全部细节 |
| 巩固区 | `signals-archive/consolidating/` | 按月双文件：`YYYY-MM-raw.json`（原始完整保留，仅供人工查阅）+ `YYYY-MM-summary.md`（聚类合并摘要，合成时只读此文件） |
| 长期记忆 | `signals-archive/long-term/` | 按年双文件：`YYYY-raw/`（原始按月保留，仅供人工查阅）+ `YYYY-summary.md`（跨月合并摘要，合成时只读此文件） |
| 图式记忆 | `signals-archive/schemas/` | established 超6个月的模型沉淀，不遗忘 |

## 六、矛盾处理三类型

### 1. 时间矛盾（认知演化）

**定义**：同一个人在不同时间表现出相反的行为/态度。

**处理方式**：这是成长的核心证据。不做"解决"，做"追踪"。

```
记录格式：
- 矛盾内容：[以前的表现] vs [后来的变化]
- 时间跨度：[日期A] ~ [日期B]
- 解读：[这个变化可能反映了什么认知层面的转变]
- 当前状态：已解决（新模式替代旧模式）/ 进行中（两种表现交替出现）
```

### 2. 领域矛盾（情境边界）

**定义**：同一个人在不同场景中表现出相反的行为/态度。

**处理方式**：标注为认知模式的情境边界条件。

```
记录格式：
- 矛盾内容：在 [场景A] 表现为 X，在 [场景B] 表现为 Y
- 解读：[这个矛盾揭示了什么情境依赖性]
- 对心智模型的影响：[相关模型需要标注"在X场景下失效"]
```

### 3. 本质张力（性格特征）

**定义**：两个持久存在的、相互拉扯的内在倾向。

**处理方式**：标记为性格的核心特征，不是要"解决"的"问题"。

```
记录格式：
- 张力内容：[倾向A] vs [倾向B]
- 持续时间：[首次观察到至今]
- 解读：[这个张力如何塑造了此人的独特性]
```

## 七、质量自检清单

画像合成完成后，逐项检查：

| 检查项 | 通过标准 | 不通过处理 |
|--------|---------|-----------|
| 模型数量适配 | ≤ 对应成熟度的最大模型数 | 合并相似模型，删除证据不足的 |
| 每个模型有局限 | 明确写出失效条件 | 补充局限描述 |
| 矛盾是否保留 | 至少检查是否有被忽略的矛盾 | 回顾信号，补充遗漏的矛盾 |
| 置信度诚实 | 不超过成熟度对应上限 | 降低过高置信度 |
| 成长轨迹完整 | 每个模型有首次出现时间 | 补充时间线信息 |
| 观察盲区标注 | 至少指出1个盲区 | 分析领域覆盖，补充盲区 |
| 观察者偏差标注 | 分析并标注偏差方向 | 补充偏差分析 |
| 诚实边界 | 至少2条具体局限 | 补充诚实边界 |
| 时间矛盾处理 | 矛盾被追踪而非被"解决" | 修正矛盾描述方式 |
| 语气检查 | 读起来像传记笔记，不是数据分析 | 重写过于机械的段落 |

**迭代上限**：质量自检最多2轮。2轮后仍有不通过项，在诚实边界中标注，交付当前版本。
