# 成长.skill

__通过日常观察，追踪任何人的认知成长轨迹。__

一个帮父亲追踪宝宝认知发育的工具，一个帮子女记录老人认知状态的档案，一个帮写作者追踪自身思维模式变化的方法。growth-skill 给所有想记录「一个人如何成长」的人准备的。

[colleague-skill](https://github.com/titanwings/colleague-skill) 把离职同事蒸馏成 AI Skill。  \
[nuwa-skill](https://github.com/alchaincyf/nuwa-skill) 把名人的认知框架蒸馏成可调用的视角。  \
growth-skill 把日常观察变成持续演化的认知画像。

## 效果示例

```
用户      ❯ 追踪 Leo，我的儿子，深度追踪
          （需要提供：性别、出生日期）
          关系：儿子
          追踪模式：深度

Growth    ✅ 已开始追踪 Leo（男，2024-06-15）。
          直接跟我说观察到的事情就行，比如"今天Leo第一次站起来了"。
          每周一早上会提醒你做一次成长合成。
```

日常记录——零摩擦，自由文本，不需要任何格式：

```
用户      ❯ 今天Leo碰到不会的拼图，先自己试了5分钟才来找我

Growth    记下了。注意到一件事：这是第二次出现"遇到困难先自己试"了，
          上一次是 4月2日的杯子事件。一个模式可能正在浮现。
          如果上面的判断不太对，直接告诉我就好。
```

```
用户      ❯ 奶奶今天说年轻人太急了，什么都想要快的

Growth    记下了。这是关于奶奶的第 8 条记录。
          "时间应该慢下来"这个价值表达已经在不同场合出现了 3 次，
          可能是她一个比较稳定的价值观。
```

积累一段时间后，生成认知画像：

```
用户      ❯ 演化 Leo

Growth    Leo 的认知画像已更新。

          📊 当前状态：developing，47 条观察，3 个心智模型

          ✨ 本次变化：
          - "先试再求助" 从 emerging 升级为 established
          - 新发现矛盾信号：以前怕生，最近主动跟邻居打招呼了

          ⚠️ 值得关注：
          - 社交领域记录偏少，建议多留意 Leo 与他人互动的场景

          完整画像见 data/leo-a3f2/profile.md
```

生成成长叙事——像传记作家写给读者的信：

```
用户      ❯ Leo的成长报告

Growth    # Leo 的成长记录

          ## 成长概览
          从 2026年1月 到 2026年4月，Leo 的认知世界经历了以下变化。

          ## 第一个认知转折（2月）
          "今天碰到不会的拼图，先自己试了5分钟才来找我。"
          这是 **"先试再求助"** 的种子信号。到 4月，这个模式在学习探索
          和社交互动两个领域反复出现，升级为已确立的心智模型。

          ## 正在浮现的模式（最近）
          "以前怕生，现在主动跟人打招呼了"
          这个矛盾信号目前还是种子状态，需要更多观察来确认是一次性变化
          还是认知转变的前兆。

          ## 你可能忽略的维度
          情绪反应领域的记录相对较少。Leo 受挫时的反应模式目前缺乏数据。

          ## 整体趋势
          Leo 正在从一个"遇到困难立刻求助"的模式转向"先独立尝试"。
          这个转变在多个领域同步出现，可能是认知能力整体跃升的信号。
```

这不是数据报告，是认知成长档案。每条观察都是一颗种子，画像合成时让种子发芽、生根、长成心智模型。

```
npx skills add .
```

### 开始使用

三步上手：追踪 → 记录 → 查看。

**1. 追踪一个人**

```
> 追踪 Leo，我的儿子，深度追踪
> 追踪 奶奶
> 追踪 自己
```

**2. 记录观察**

直接说观察到的事情，自由文本，不需要任何格式：

```
> 今天Leo第一次自己站起来了
> 记录：他碰到烫的东西缩回手，但第二天又去摸
> 奶奶今天说年轻人太急了
```

**3. 查看成长**

```
> 演化 Leo                # 生成/更新认知画像
> Leo的成长报告            # 成长叙事
> 复盘奶奶                # 成长复盘
> 看看状态                # 所有追踪对象总览
```

## 核心特性

### 零摩擦记录

用户写自由文本，不需要结构化输入。系统自动判断输入意图——你说「今天Leo...」，它就知道这是一条观察；你说「成长报告」，它就开始复盘。

### 八类认知信号

每条日常观察会被提取为以下类型的认知信号：

| 信号 | 说明 | 示例 |
| --- | --- | --- |
| __心智模型种子__ | 可能的认知框架雏形 | "遇到问题先自己试" |
| __决策启发式种子__ | 决策模式雏形 | "面对选择总是选更安全的" |
| __价值观表达__ | 对是非的判断 | "他觉得分享比独占更快乐" |
| __反模式__ | 明确拒绝的行为 | "他非常讨厌被强迫做某事" |
| __表达指纹__ | 认知框架偏好 | "描述事物喜欢用比喻" |
| __矛盾信号__ | 与已有信号矛盾 | "以前怕生，现在主动打招呼了" |
| __里程碑__ | 重要人生事件 | "开始上幼儿园了" |
| __情绪模式__ | 情绪反应模式 | "受挫时会先沉默再爆发" |

信号会经历完整生命周期：`seed → emerging → established → evolving → dormant`。数据越多，验证越严格——不会把一次观察包装成「已确立的认知模式」。

### 矛盾即成长

时间矛盾（以前怕生，现在主动打招呼了）不是错误，是认知成长的核心证据。系统追踪矛盾而非「解决」矛盾，让画像呈现真实的认知演化过程。

### 年龄感知分析

出生日期是必填项，因为同一条行为，3 岁宝宝和 30 岁成人的认知意义完全不同。系统根据当前年龄自动调整分析框架的侧重方向和期望基准。

### 自适应验证

数据少时宽容（宁可漏掉也不误判），数据多时严格（宁缺毋滥）：

| 观察数量 | 最大模型数 | 置信度上限 | 成熟度 |
| --- | --- | --- | --- |
| < 10 | 2 | 0.4 | early |
| 10-50 | 4 | 0.6 | developing |
| 50-100 | 6 | 0.8 | mature |
| > 100 | 7 | 1.0 | mature |

### 画像包含什么

生成认知画像时，从信号中构建完整的认知档案：

- **核心心智模型** — 此人用什么框架看世界
- **决策启发式** — 此人怎么做判断
- **表达模式** — 此人的认知框架偏好
- **价值观与反模式** — 此人追求什么、拒绝什么
- **成长时间线** — 认知变化的关键节点
- **矛盾与张力** — 成长的核心证据
- **观察盲区** — 哪些维度缺乏数据
- **观察者偏差** — 系统性的记录倾向
- **诚实边界** — 画像做不到什么

__一个不告诉你局限在哪的画像，不值得信任。__

## 工作原理

growth-skill 的核心是一个模拟人脑记忆周期的信号系统：

```
日常观察
  ↓ Phase 1: 轻量信号提取
认知信号（seed → emerging → established → evolving → dormant）
  ↓ Phase 3: 画像合成（"睡眠巩固"）
认知画像（profile.md）
  ↓ Phase 4: 成长复盘
成长叙事
```

### 遗忘曲线

**遗忘不是删除，是衰减。** 信号遵循 Ebbinghaus 遗忘曲线：

```
retention = e^(-t / stability)
```

不同信号类型遗忘速度不同：里程碑像「闪光灯记忆」（180 天稳定性），情绪模式随情境变化（30 天）。重复出现会增强稳定性，模拟间隔重复效应。

### 四层记忆架构

信号在四层记忆中流转，模拟人脑从感知到图式的完整路径：

| 层 | 类比 | 存储 | 行为 |
| --- | --- | --- | --- |
| __工作记忆__ | 前额叶皮层 | `signals.jsonl` | 容量 ~200 条，保留全部细节 |
| __巩固区__ | 海马体转移中 | `consolidating/` | 按月归档，聚类合并相似信号 |
| __长期记忆__ | 新皮层 | `long-term/` | 按年归档，保留信号骨架 |
| __图式记忆__ | 认知图式 | `schemas/` | 已确立模式永久沉淀，不遗忘 |

被遗忘的信号被新观察重新匹配时，会触发**再巩固**——从归档召回，结合新证据重新评估。这模拟了人脑「突然想起某件事」的体验。

## 仓库结构

```
growth-skill/
├── SKILL.md                        # 主 Skill 定义（核心编排逻辑）
├── references/
│   ├── growth-framework.md         # 方法论文档（信号提取、遗忘曲线、质量自检）
│   └── profile-template.md         # 认知画像输出模板
└── data/                           # 运行时数据（不提交）
    ├── registry.json               # 所有追踪对象索引
    └── [person-id]/
        ├── person.json             # 元信息
        ├── observations/           # 原始日记（YYYY-MM-DD.md）
        ├── signals.jsonl           # 工作记忆（活跃信号）
        ├── signals-archive/        # 归档信号
        │   ├── consolidating/      # 巩固区（按月）
        │   ├── long-term/          # 长期记忆（按年）
        │   └── schemas/            # 图式记忆（永久沉淀）
        ├── profile.md              # 当前认知画像
        └── profile-history/        # 画像历史快照
```

完整方法论见 `references/growth-framework.md`。

## 背后的故事

真正让你日复一日观察、关心其成长的，是你的孩子、父母、朋友、甚至你自己。这些人的认知变化，没有任何工具在系统追踪。

每一条日常观察——「今天第一次自己站起来了」「他说分享比独占更快乐」「以前怕生，现在主动打招呼了」——都是一颗认知种子。但如果你不记录，这些种子就散落在记忆里，无法生根发芽。

growth-skill 做的事很简单：帮你种下这些种子，然后让它们长成认知画像。

__growth-skill 不创造认知，它发现认知。__
